# SEO Content Skill

Produce content that ranks and converts — structured around search intent, keyword strategy, and on-page best practices.

## Workflow

1. **Keyword & intent analysis**: primary keyword, search intent (informational/navigational/transactional/commercial)
2. **Competitor gap check**: what top-ranking pages cover (and what they miss)
3. **Content brief**: outline with H2/H3 structure, word count, internal links
4. **Write the content** using the SEO structure below
5. **On-page checklist**: meta title, meta description, image alt text, internal links

---

## Search Intent Types

| Intent | User goal | Content type |
|---|---|---|
| Informational | Learn about X | Blog post, guide, explainer |
| Navigational | Find a specific site | Homepage, brand pages |
| Transactional | Buy/hire/sign up | Landing page, product page |
| Commercial investigation | Compare before buying | Comparison, review, "best X" |

**Always match content type to intent** — informational content won't rank for transactional keywords.

## On-Page SEO Checklist

### Title tag (H1 / meta title)
- Include primary keyword, preferably near the start
- 50–60 characters for meta title
- H1 can be slightly longer and more descriptive
- One H1 per page

### Meta description
- 150–155 characters
- Include primary keyword naturally
- Include a benefit or CTA ("Learn how to...", "Find out why...")
- Not a ranking factor but affects CTR

### Heading structure
```
H1: [Primary keyword + topic]
  H2: [Subtopic 1 — secondary keyword]
    H3: [Specific point]
    H3: [Specific point]
  H2: [Subtopic 2]
    H3: ...
  H2: FAQ (if applicable)
```

### Keyword density
- Primary keyword: 1–2% density (don't force it)
- Include in: H1, first 100 words, at least 1 H2, naturally throughout
- LSI/semantic keywords: use related terms, not just the exact phrase

### Internal linking
- Link to 2–4 relevant internal pages
- Use descriptive anchor text (not "click here")
- Include at least one link from a high-authority page to this new content

### Content quality signals
- **Word count**: match or exceed top-ranking competitors (check SERPs)
- **Originality**: add data, examples, or angles competitors miss
- **Freshness**: include current year, recent data, updated statistics
- **Multimedia**: suggest where images, tables, or videos would help

## Blog Post Structure

### Introduction (100–150 words)
- Open with the problem or question the reader has
- Include primary keyword in first 100 words
- Preview what they'll learn (not clickbait — deliver on the promise)
- Avoid starting with "In this article..."

### Body sections
- Each H2 = one complete subtopic
- Lead each section with the most important information (inverted pyramid)
- Use bullet points and numbered lists where appropriate
- Include 1 internal link per 500 words approximately

### Conclusion (100–150 words)
- Summarize key takeaways (3–5 bullets or short paragraphs)
- Answer the main keyword question directly if not done above
- CTA: what should they do next?

### FAQ section (optional but recommended)
- Target "People Also Ask" questions from Google
- Answer each question in 2–4 sentences
- Use H3 for each question

## Content Cluster Strategy

For building topical authority:

```
Pillar Page (broad topic, 2,000–4,000 words)
├── Cluster Post 1 (specific subtopic, 1,000–2,000 words)
├── Cluster Post 2 (specific subtopic)
├── Cluster Post 3 (specific subtopic)
└── ... (8–15 cluster posts)
```

All cluster posts link to the pillar page. Pillar page links to all cluster posts. This builds topical authority faster than random posts.
