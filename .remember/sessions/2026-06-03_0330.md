## Contexto no obvio

- **_elementor_data y REST API de WordPress**: hay un conflicto sistémico entre Yoast SEO y Elementor cuando se actualiza una página vía REST API. Yoast dispara `apply_filters('the_content')` al indexar, Elementor intenta renderizar desde `_elementor_data`, y si el JSON está malformado explota con TypeError. El resultado: `update_additional_fields_for_object()` NUNCA se ejecuta y el meta NO se guarda. La única solución fiable es `$wpdb->update()` directo (bypass completo de hooks).
