## Contexto no obvio

- **`_elementor_data` accesible vía REST API** — descubierto hoy. Meta registrado con `show_in_rest: true` en esta instalación. El patrón `wrap_elementor(html)` en `output/agency/wp_update_elementor_pages.py` es la referencia. **Todas las páginas nuevas deben crearse así**, no via `content` normal (que cae en Text Editor widget).

- **Bug CSS dobles llaves** — ocurre cuando CSS se genera en Python con `.replace()` en vez de f-string. Si el CSS tiene `{{` o `}}` en vez de `{` y `}`, los estilos no aplican en el navegador. El fix está en `wp-page-rd.md` y en `wp_fix_service_css.py`.

- **`/servicios-google-ads/` publicada con datos genéricos** — NO compartir con Tecniclima hasta tener datos reales de Kevin. Tiene benchmarks de relleno en la sección de resultados.

- **Bóveda tiene placeholder `<!-- MAILERLITE_FORM -->`** en dos sitios: en la sección de newsletter y en la tarjeta featured del hero. Cuando Kevin dé el embed de MailerLite, localizar estas dos cadenas en el `_elementor_data` y sustituirlas.

- **Menú único (ID 8)** — la instalación solo tiene 1 menú registrado. Las links legales del footer son hardcoded en Elementor, no son un menú de WP Admin.

- **Manual de Claudia (ID 6805)** — está publicado (status: publish) aunque el inventario inicial lo mostraba como draft. Verificado por API directa.

- **Google Ads API env vars** — el patrón de naming acordado: `GOOGLE_ADS_DEVELOPER_TOKEN`, `GOOGLE_ADS_CLIENT_ID`, `GOOGLE_ADS_CLIENT_SECRET`, `GOOGLE_ADS_REFRESH_TOKEN`, `GOOGLE_ADS_LOGIN_CUSTOMER_ID`, `GOOGLE_ADS_TECNICLIMA_CUSTOMER_ID`.
